# Project Meowstik — Product Direction Note

## Core idea

Project Meowstik can evolve in two complementary directions at once:

1. **agentic productivity / workspace system**
2. **advanced companion platform**

Those do not need to be the same SKU forever, but they can share infrastructure.

## Companion-platform direction

The relevant comparison is not merely "chatbot."
It is closer to:

- Replika replacement
- device-present assistant
- persistent companion identity
- cross-device agent presence

Possible surfaces:

- smartphone
- watch
- desktop
- robot pet
- telepresence bot
- car center console

## Shared infrastructure with Greplit

What Greplit and Meowstik can share:

- agent orchestration
- memory
- execution logs
- MCP integrations
- identity and session state
- local-first runtime patterns
- device-aware UX

## Important product distinction

Greplit is about **building and operating software workspaces**.

Meowstik-as-companion is about:

- presence
- continuity
- emotional UX
- task help
- ambient device integration

The overlap is architectural, not necessarily branding.

## Adult-conversation note

Provider behavior should be treated as **provider-scoped capability**, not a core assumption.

If the product ever supports more permissive companion modes, that should be:

- optional
- provider-specific
- policy-gated
- clearly separated from the default experience

The default Meowstik value proposition does not need explicit-adult content to be compelling.

## Strategic takeaway

Greplit can be the **builder / workspace product**.

Meowstik can be the **persistent agent / companion platform**.

They can reinforce each other if they share the same underlying system for memory, logs, tools, integrations, and multi-device presence.
